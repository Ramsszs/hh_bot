# hh.kz Resume Touch Bot

Автоматически поднимает резюме в поиске на hh.kz раз в 4 часа.

## Переменные окружения

| Переменная | Обязательная | Описание |
|---|---|---|
| `HH_COOKIE` | да | Полная строка куки из браузера |
| `HH_XSRF` | да | Значение `_xsrf` куки (для заголовка `X-Xsrftoken`) |
| `HH_RESUME_HASH` | да | Хэш резюме (из URL резюме на hh.kz) |
| `HH_DOMAIN` | нет | Домен hh (по умолчанию `astana.hh.kz`) |
| `HH_USER_AGENT` | нет | User-Agent браузера |
| `RUN_MODE` | нет | `once` (по умолчанию) или `loop` |

## Как получить куки

1. Открой hh.kz в браузере, залогинься
2. Открой DevTools (F12) → Network
3. Нажми «Поднять в поиске» на своём резюме
4. Найди запрос `touch` в Network
5. Правый клик → Copy → Copy as cURL
6. Из cURL возьми значения `Cookie` и `_xsrf`

## Деплой на Railway

### Вариант 1: Cron (рекомендуется)

1. Создай новый проект на Railway, подключи этот репозиторий
2. В настройках сервиса задай переменные окружения
3. Settings → Cron Schedule: `0 */4 * * *`
4. `RUN_MODE` оставь `once` (или не задавай)

Railway будет запускать контейнер каждые 4 часа, скрипт сделает запрос и завершится.

### Вариант 2: Loop (если Cron недоступен)

1. Задай `RUN_MODE=loop`
2. Скрипт будет работать непрерывно, отправляя запрос каждые 4 часа

## Когда протухнут куки

При 403 ответе в логах будет сообщение:
```
403 Forbidden — cookies/xsrf expired. Update HH_COOKIE and HH_XSRF in Railway env vars.
```

Повтори шаги из раздела «Как получить куки» и обнови переменные в Railway.
